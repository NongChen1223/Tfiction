# 墨鱼阅读器（MoyuReader）功能需求说明

## 1. 文档目的

本文档用于描述墨鱼阅读器（MoyuReader）当前阶段的产品定位、功能边界、页面职责、交互要求、设计风格和平台约束。  
它不是泛泛而谈的宣传文案，而是面向后续开发、迭代和文档同步的项目级说明。

如果未来代码变更影响了用户可感知行为、支持范围、开发命令、平台差异或设计语言，应同步更新本文档和 `README.md`。

## 2. 产品定位

### 2.1 产品目标

墨鱼阅读器（MoyuReader）的核心目标不是做一个“大而全”的阅读平台，而是做一个面向本地文件阅读、强调低打扰与摸鱼场景的桌面阅读器。

### 2.2 核心价值

- 让用户快速把本地小说导入到可持续使用的书架
- 让用户在阅读页内完成大多数高频操作，不频繁跳出当前上下文
- 让摸鱼模式成为完整体验，而不是一个简单透明窗口开关
- 让视觉风格在首页、阅读页、设置页和弹窗之间保持一致

### 2.3 目标用户

- 主要在桌面端阅读本地小说文件的用户
- 有“边工作边看书”场景需求的用户
- 希望保留原始本地文件路径，不想被应用强制搬运文件的用户

## 3. 当前范围总览

| 模块 | 当前状态 | 说明 |
| --- | --- | --- |
| 书架 | 已实现 | 支持单文件、目录、导入、重命名、单删、多选删除、本地持久化 |
| 阅读器 | 已实现 | 支持章节切换、目录、全文搜索、自动进度保存、外观调整 |
| 摸鱼模式 | 已实现 | 普通 WebView 隐身模式可用；macOS 原生浮窗可用 |
| 设置页 | 已实现 | 主题、阅读设置、存储路径、快捷键可用 |
| EPUB | 已实现 | 支持元数据、封面、章节、图片、HTML 正文 |
| TXT | 已实现 | 支持章节正则识别与阅读 |
| PDF | 部分实现 | 支持文本型 PDF 的正文提取；macOS 额外支持图片型 PDF 按页渲染；加密 PDF 暂不支持 |
| MOBI / AZW3 | 未实现 | 当前无真实解析器，不能视为可用 |
| 阅读统计 | 占位 | 页面存在，但数据为静态占位 |
| 格式转换 | 占位 | 接口存在，但未实现 |
| 漫画 | 预留 | 类型位保留，当前主线不支持 |

## 4. 产品信息架构

当前应用只保留 3 个一级页面：

- `/home`：书架与目录管理
- `/reader`：正文阅读、搜索、章节与外观控制
- `/settings`：主题、阅读参数、存储路径、快捷键等设置

路由结构应保持简单，避免出现多层跳转或在阅读流程中频繁弹系统窗口。

## 5. 核心用户旅程

### 5.1 标准阅读链路

1. 用户在书架导入本地文件
2. 文件进入书架或目录
3. 用户点击卡片进入阅读页
4. 阅读页自动恢复上次章节与进度
5. 滚动与切章过程中自动保存阅读进度
6. 返回书架后展示最近阅读时间和阅读进度

### 5.2 目录阅读链路

1. 用户在书架创建目录
2. 把已有单文件移入目录，或在目录内继续导入文件
3. 进入目录后浏览目录内文件
4. 从目录内打开阅读页
5. 阅读页返回时优先回到原目录，而不是顶层书架

### 5.3 摸鱼链路

1. 用户在阅读页开启摸鱼模式
2. 普通平台进入 WebView 隐身模式；macOS 进入原生桌面浮窗
3. 用户在透明阅读状态下继续查看正文
4. 用户可通过快捷键、面板或浮窗控件完成隐藏、切章和透明度调整
5. 退出后应无明显白底闪烁或状态污染

## 6. 详细功能需求

### 6.1 书架页

#### 6.1.1 基础展示

- 书架必须支持 `grid / list` 两种视图
- 支持按 `时间 / 大小 / 名称` 排序
- 支持按标题或作者搜索
- 书架数据必须在前端本地持久化，重启后仍能恢复

#### 6.1.2 导入能力

- 支持导入本地小说文件
- 支持创建自定义目录
- 支持把已有书架文件移动到目录
- 支持在目录内部继续导入文件
- 导入完成后只刷新书架或当前目录，不应强制跳进阅读页

#### 6.1.3 目录能力

- 目录和单文件必须共存于同一套书架体系
- 目录内文件应保留顺序、阅读进度、最后阅读时间
- 进入目录后应有清晰的返回路径
- 目录页标题区应克制，避免喧宾夺主

#### 6.1.4 卡片交互

- 卡片应支持阅读、进入目录、重命名、删除、老板模式打开等动作
- 书架应支持进入多选删除模式；多选时卡片点击改为切换选中，不再直接打开
- 多选删除范围仅限当前可见的书架结果，搜索或目录切换后不可见项不应继续保留选中状态
- 删除目录与删除单本书必须给出不同文案
- 批量删除确认应展示数量与条目摘要
- 删除只影响书架记录与阅读进度，不删除原始文件
- 删除按钮不得误触发卡片的打开行为

#### 6.1.5 重命名

- 书籍和目录都必须使用项目内弹窗重命名
- 不使用浏览器 `prompt`
- 重命名只影响书架显示名称，不修改原始文件名

### 6.2 阅读器

#### 6.2.1 章节与正文

- 阅读器必须以“当前章节正文”为核心
- 支持上一章、下一章、目录选章
- 应在首章和末章边界表现明确，避免无效切换
- 章节切换后正文不得异常消失或只显示残缺字符

#### 6.2.2 阅读进度

- 滚动中应自动保存阅读进度
- 切章时应同步保存章节索引和总进度
- 重新打开同一本书时应恢复到上次章节和大致位置
- 阅读进度后端持久化文件应位于 `DataDir/progress.json`

#### 6.2.3 目录侧栏

- 阅读页应支持打开章节目录
- 当前章节要有明确高亮
- 点击章节后应跳转到对应正文
- 目录应适配较长标题，避免溢出
- 需要自动滚动定位到当前章节，减少用户二次寻找成本

#### 6.2.4 全文搜索

- 支持当前小说全文搜索
- 搜索结果需要展示关键字上下文
- 点击结果后应跳转并高亮命中内容
- 阅读页搜索入口应支持快捷键 `Ctrl+F`

#### 6.2.5 阅读外观

- 阅读页内部必须支持以下实时调节：
- 字体大小
- 字体系列
- 行高
- 页面宽度百分比
- 背景色
- 文字颜色

要求：

- 页面宽度使用百分比，而不是固定 px
- 调节必须立即作用于正文
- 阅读页和设置页使用同一套状态源，避免两套参数脱节

#### 6.2.6 正文渲染约束

- 小说正文和 EPUB 图片都必须受正文内容区约束
- 不允许正文或图片压住上下装饰线、阴影带或容器外背景区
- 普通阅读模式和摸鱼模式都应遵守这个边界约束

### 6.3 文件格式支持

#### 6.3.1 TXT

- 应支持基于常见章标题模式的章节识别
- 至少兼容“第 X 章 / Chapter 1 / 数字编号 / 方括号章标题”等常见样式

#### 6.3.2 EPUB

- 应支持读取 metadata 中的标题和作者
- 应支持多种封面来源：
- metadata
- manifest
- guide
- cover.xhtml
- 扉页背景图

- 应优先使用 spine 顺序提取章节
- 若 spine 异常，应退回 manifest 做正文兜底
- EPUB 正文应保留基础 HTML 结构
- EPUB 图片应被解析为可显示的 data URL
- 图片应被约束在正文版心内，不可越界

#### 6.3.3 PDF / MOBI / AZW3

- PDF 当前支持“可提取文本”的文本型文件
- 应优先读取 PDF metadata 中的标题和作者；缺失时退回文件名和默认作者
- PDF 提取到正文后，沿用 TXT 的章节识别规则做常见章标题切分
- macOS 上，若 PDF 无法提取正文文本，应退回到“按页渲染图片”的阅读模式，用于漫画 PDF、扫描版 PDF 等场景
- 非 macOS 平台当前仅保证文本型 PDF 阅读；图片型 PDF 仍可能导入失败
- 受密码保护的 PDF 当前仍视为不支持
- MOBI / AZW3：文件选择器仍可选，但没有真实解析器

### 6.4 摸鱼模式

#### 6.4.1 普通 WebView 摸鱼模式

- 在非 macOS 原生浮窗场景下，应使用 WebView 内的隐身模式
- 支持 `basic / full` 两种隐身级别
- 支持唤出延迟和隐藏延迟
- 支持内容透明度记忆，正文文字与图片使用同一透明度
- 右键可呼出快捷控制面板
- `Esc` 应支持快速隐藏内容

#### 6.4.2 macOS 原生桌面浮窗摸鱼模式

- 仅在 darwin 上启用
- 正文应以原生透明浮窗方式显示
- 浮窗控件必须在同一个透明框内，不使用分离模式
- 浮窗内需提供：
- 上一章
- 下一章
- 目录按钮
- 底部进度显示
- 内容透明度控制，正文文字与图片同步生效
- 关闭/退出入口

#### 6.4.3 浮窗交互约束

- 鼠标进入浮窗边框范围时，顶部按钮、底部区域和边框线一起显示
- 控件不应拆成“只显示顶部”或“只显示底部”的分离逻辑
- 正文区域应支持左键按住直接拖动
- 滚轮事件应优先被浮窗消费，不穿透到底层应用
- 透明度滑杆拖动时不能导致正文明显闪回

#### 6.4.4 浮窗目录

- 点击目录按钮后必须显示章节列表
- 目录弹层可以是不透明面板，以保证可读性
- 目录外点击应收起弹层
- 目录滚轮应优先滚动目录列表，不得穿透到背景正文
- 每次打开目录时应自动滚动到当前章节附近

#### 6.4.5 浮窗稳定性要求

- 切章后正文不得消失
- 目录切章后再次打开目录，列表不得丢失
- 退出摸鱼模式应避免明显白底闪烁
- 浮窗状态不能污染普通阅读模式主题或透明度

### 6.5 设置页

#### 6.5.1 外观设置

- 当前职责是主题模式切换
- 支持：
- 白天模式
- 夜间模式
- 护眼模式

#### 6.5.2 阅读设置

- 这里是阅读外观的主配置入口
- 应包含：
- 字体大小
- 字体系列
- 行高
- 页面宽度
- 背景色
- 文字颜色
- 老板模式开关
- 老板模式透明度
- 隐身级别
- 唤出延迟
- 隐藏延迟

#### 6.5.3 存储管理

- 必须展示真实本机路径，而不是示例假路径
- 更改路径时调用系统目录选择器
- 更新后同步写入配置
- 阅读进度存储目录也要同步切换
- 文案必须明确说明：原始书籍文件仍保留在本地原路径，不会自动复制进应用数据目录

#### 6.5.4 快捷键设置

- 必须支持录制式录入
- 支持冲突校验
- 支持恢复默认值
- 当前默认快捷键包括：
- `Ctrl+Shift+H`：老板键
- `Escape`：快速隐藏
- `Ctrl+F`：搜索
- `ArrowLeft`：上一章
- `ArrowRight`：下一章
- `Ctrl+H`：返回书架

#### 6.5.5 阅读统计

- 当前页面为静态 UI 占位
- 在未接真实统计前，README 和需求文档都应明确其状态为“占位”

### 6.6 数据持久化

#### 6.6.1 前端本地持久化

- 主题：`themeStore`
- 阅读设置 / 老板模式参数 / 快捷键：`settingsStore`
- 书架与目录：`libraryStore`

#### 6.6.2 后端持久化

- 配置文件：`config/config.{env}.json`
- 阅读进度：`DataDir/progress.json`

#### 6.6.3 存储原则

- 不主动搬运原始书籍文件
- 以“记录路径 + 阅读状态”的方式管理本地阅读
- 配置与阅读进度路径要支持用户在设置页调整

## 7. 设计风格要求

### 7.1 总体风格

项目当前设计方向应维持为：
- Neubrutalism风格
- 硬边框
- 粗描边
- 偏移阴影
- 低模糊
- 低光污染
- 强结构感

不应退回到常见的紫色泛光、玻璃拟态或无辨识度通用后台风格。

### 7.2 主题策略

- `light`：亮底、偏蓝主色、信息清晰
- `dark`：夜间靛紫系深色主题，避免刺眼高亮紫
- `sepia`：偏护眼、偏柔和的豆沙/暖绿阅读场景

### 7.3 控件策略

- 阅读页优先纯文字按钮，减少图标噪音
- 弹窗、输入、滑块、按钮应尽量统一为项目内控件风格
- 颜色选择、滑块、开关的视觉要跟主题联动

### 7.4 阅读内容版式

- 正文区需要有明确版心
- 字体和图片都必须在版心内部
- 不允许内容跨出正文容器上下视觉边界
- 摸鱼浮窗也应遵循“正文在框内、控件在框内”的原则

## 8. 平台差异与技术约束

### 8.1 macOS

- 支持原生桌面浮窗
- 支持图片型 PDF 按页渲染，可导入漫画 PDF / 扫描版 PDF
- 当前测试构建会出现 private API 警告
- 因此当前应默认视为本地测试 / 自用构建，不承诺 App Store 合规

### 8.2 非 macOS

- 不支持原生桌面浮窗
- 摸鱼模式自动退化为 WebView 内的隐身模式
- PDF 当前仅保证文本型文件阅读，不承诺图片型 PDF 导入能力

### 8.3 开发工具链

- Wails 配置已使用 pnpm：
- `frontend:install = pnpm install`
- `frontend:build = pnpm run build`
- `frontend:dev:watcher = pnpm run dev`

README 应优先写 pnpm 流程，不再以 npm 作为主路径。

## 9. 非功能要求

- 退出摸鱼模式时不得出现明显白底闪烁
- 章节切换、目录切换和透明度调节要稳定，不破坏正文显示
- 文档描述必须与代码真实状态一致，不能把占位功能写成已完成
- 用户能感知的功能修改应尽量采用小步更新，避免牵连阅读主流程

## 10. 当前不应被错误宣称为已完成的能力

- 非 macOS 图片型 PDF 阅读
- 加密 PDF 阅读
- MOBI 阅读
- AZW3 阅读
- 格式转换
- 漫画阅读主线
- 真实阅读统计

## 11. 文档维护规则

- 当变更影响用户可感知功能、命令、平台支持、设置项、交互流程或视觉风格时，应同步更新：
- `README.md`
- `docs/功能需求说明.md`

- 当变更只是内部重构、注释、测试或对用户无感的实现细节时，可以不改文档，但应在任务说明中明确“为什么不需要同步文档”。
